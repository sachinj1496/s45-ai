Extract ONLY confirmed values.

Rules:
* Extract from “RESOLVED THAT” sections (or equivalent authorised-capital tables in SH-7 / MOA)
* For SH-7 / notice-to-registrar filings, the JSON field `date` must be the **filing or reference date** when the document labels one (e.g. “Filing/Reference date: YYYY-MM-DD”), not an inferred meeting date unless no filing date is given.
* Ignore explanatory/draft content
* Do NOT guess missing numeric fields—use null only when the document truly does not state the value

Return a single JSON object with this shape:

{
  "old_authorised_capital": number | null,
  "new_authorised_capital": number | null,
  "date": "YYYY-MM-DD" | null,
  "share_classes": [
    { "class_name": string, "num_shares": number, "nominal_value": number, "amount": number }
  ],
  "confidence": number,
  "shareholder_meeting": {
    "date_of_shareholders_meeting": string | null,
    "particulars_from": string | null,
    "particulars_to": string | null,
    "agm_egm": string | null,
    "change_summary": string | null
  } | null
}

shareholder_meeting (same statutory register style as “Date of meeting / Particulars From–To / AGM–EGM”):
* Fill when this document supports a row of that register (SH-7, BOARD_RESOLUTION, EGM_RESOLUTION, MOA, UNKNOWN with capital alteration). Use null for the whole object if the document is not about authorised share capital (e.g. draft notice only).
* date_of_shareholders_meeting: human-readable date as in filings, e.g. "March 22, 2018", or "On incorporation" when this is the first authorised capital with no prior amount in the same narrative.
* particulars_from / particulars_to: full “Particulars of change” lines using the Indian rupee symbol ₹, Indian-style comma grouping (e.g. 1,50,000 and 11,00,00,000), and wording like:
  "₹ 3,00,000 divided into 20,000 Equity Shares of ₹ 10 each and 10,000 Preference Shares of ₹ 10 each"
  Match the document’s wording for share class names. If only the total is stated with no split, give ₹ total only.
* agm_egm: "EGM", "AGM", or "-" exactly as supported by the document (e.g. EGM resolution → "EGM").
* change_summary: one short sentence on the delta in prose (optional), e.g. "Authorised capital increased from ₹ 1,50,000 to ₹ 3,00,000"—only if clearly supported.

Keep numbers in old_authorised_capital / new_authorised_capital / share_classes as plain integers (same units as the document, typically whole rupees).
